# 🏠 Automation Test

본 프로젝트는 **Playwright**를 활용하여 **호갱노노(HoGangNoNo)** 서비스의 핵심 사용자 기능을 자동화한 테스트 프로젝트입니다. 자동화 구조는 테스트 유지보수성과 확장성을 높이기 위해 **Page Object Model(POM)** 기반으로 설계되었으며, 반복적인 시나리오를 안정적으로 검증할 수 있도록 구성되었습니다.

---

## 📌 프로젝트 개요

본 프로젝트는 실제 사용자가 서비스를 이용하는 핵심 플로우 중심으로 자동화를 구축했습니다.

* 로그인 이후 주요 기능 사용 흐름 자동화
* 단지 검색 기능 검증
* 알림 등록/삭제 기능 검증
* 핵심 사용자 경험(UX) 흐름을 우선적으로 자동화

테스트는 사용자의 실제 행동을 기준으로 설계되었으며, 서비스 변경에 유연하게 대응할 수 있도록 구조화했습니다.

---

## 📁 프로젝트 구조

```
e2e/
 ├─ pages/                    # 페이지 객체 모음
 │    ├─ login                # 로그인 
 │    │   └─ login.page.ts
 │    ├─ search               # 단지 검색 
 │    │   └─ search.page.ts
 │    └─ notification         # 알림 설정
 │        └─ notification.page.ts
 ├─ tests/                # 테스트 시나리오
 │    ├─ login                
 │    │   └─ login.spec.ts
 │    ├─ search               
 │    │   └─ search.spec.ts
 │    └─ notification
 │        └─ notification.page.ts
 │
 └─ utils/             # 공통 테스트 설정
```

---

## 🔎 자동화한 기능 목록

### 1) 로그인 기능

* 정상 로그인 흐름 검증
* 잘못된 인증 정보 입력 시 오류 메시지 확인

### 2) 단지 검색 기능

* 검색창을 통한 단지명 검색
* 지도 렌더링 및 검색 결과 확인

### 3) 알림 설정 기능

* 알림 추가 및 삭제 동작 확인
* 관심 목록의 정상 반영 여부 확인

## 🚀 기대 효과

| 항목         | 기대 효과                |
| ---------- | -------------------- |

## 📚 결론

이 자동화 테스트 프로젝트는 Playwright 기반의 구조적 자동화를 통해 호갱노노 서비스의 핵심 사용자 흐름을 안정적으로 검증하는 것을 목표로 합니다.

* 사용자 중심 시나리오
* 유지보수 친화적인 구조
* 서비스 변경에 유연하게 대응 가능한 설계
